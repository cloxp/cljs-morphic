Clojurize me...
An experiment to see how a morphic system can be implemented using ClojureScript and om/react.
Lively Morphic proofed to be very flexible in providing a mechanism for ad-hoc content creation and scripting live objects. Mechanisms like the ObjectEditor and PartsBin to allow to create and program graphical objects and user interfaces in minutes and to share the results on web pages. There are, however, drawbacks to the implementation: - Since state of applications and the Morphic system itself is not localized but split over the entire system it turned out to be complicated to create synchronization between applications and Morphs. This is a problem since Lively's ecosystem and domain could be enhanced by providing multi-user applications and live state synchronization. - The approach of the react DOM library in particular is very interesting because it provides an efficient layer on top of the slow and complicated DOM interfaces. - The thesis of this experiment is that the way how react and Clojure handle state will simplify a Morphic implementation. Especially in terms of how state synchronization can be implemented this approach might offer several advantages over existing prototypes.
